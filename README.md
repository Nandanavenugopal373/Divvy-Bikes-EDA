# Divvy-Bikes-EDA

Project Overview

This project performs Exploratory Data Analysis (EDA) on the Cyclistic Bike Share dataset using Python. The objective of the analysis is to understand the riding behavior of casual riders and annual members by analyzing ride patterns, ride durations, bike preferences, and usage trends.

The project includes data cleaning, preprocessing, visualization, and interpretation of insights using various Python libraries.

Objectives
Analyze customer riding behavior
Compare casual riders and members
Identify rideable type preferences
Understand ride duration and usage patterns
Generate business insights from the dataset
Technologies Used
Python
Pandas
NumPy
Matplotlib
Seaborn
Jupyter Notebook
Dataset Information

The dataset used in this project is the Cyclistic Bike Share dataset.

Dataset Source

The dataset was obtained from the Divvy Bike Sharing System data repository:

https://divvy-tripdata.s3.amazonaws.com/index.html

Dataset Description

The dataset contains ride-level information including:

Ride ID
Rideable type
Start and end timestamps
Start and end station names
Start and end station coordinates
Member type (casual/member)

The dataset is used to analyze customer usage behavior and identify riding trends among different user groups.

Data Cleaning Steps
Removed duplicate records
Handled missing values
Converted date columns into datetime format
Extracted day, month, and hour features for analysis
Key Insights
Electric bikes are more preferred than classic bikes among both casual riders and members.
Members contribute a higher number of rides overall.
Casual riders generally have longer ride durations.
Ride activity varies across weekdays and weekends.
Conclusion

The analysis provides useful insights into customer behavior and bike usage trends. These insights can help improve marketing strategies, customer engagement, and operational decisions for bike-sharing services.
